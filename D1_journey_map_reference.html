Stage 5
Active Implementation
M1–M5 · weekly pulse · module-aware JIT kits · dynamic tier logic · Monday digest to Ivan
▾
Owner
Teacher (weekly pulse); Ivan (Red-site direct intervention + Monday triage)
Airtable status
Active (current session field updated weekly via pulse)
Mostly automated
Module-specific risk windows (curriculum-aware detection)
M1 (S1–S5): Student commitment to brief. Flag if Brief Analysis Sheet not submitted by S5.
M2 (S6–S10): Interview quality — weak interviews cascade to all downstream modules. Flag if interviews not scheduled by S7. Deliverable: Interview Submission (3+ interviews + notes + takeaways).
M3 (S11–S15) — HIGHEST RISK: Synthesis leap. All M3 entries auto-elevated to Yellow. Score ≤ 2 = Red, Ivan direct call within 24h. Deliverable: Insight Statement (quotes → themes → insights + HMW).
M4 (S16–S19): Ideas must connect to insights. Flag if concept maps not grounded in M3 deliverable. Deliverable: Top 3 Concept Maps.
M5 (S20–S24): Client feedback iteration. Flag if Prototype v2 not measurably different from v1. Deliverable: Prototype v1 + v2.
Tier logic (dynamic — updates automatically)
🟢 Green: On-time pulses, scores ≥ 3, deliverables submitted on pace
🟡 Yellow: 2 consecutive score ≤ 2, OR entering M3 (auto-elevated), OR 1 missed submission
🔴 Red: Score ≤ 2 in M2/M3 window, OR module deliverable missing at module end, OR 2+ consecutive missed submissions
Risk signals
Pulse score ≤ 2 for 2 consecutive weeks → Yellow tier auto-update
Pulse score ≤ 2 in M2 (S6–S10) or M3 (S11–S15) → Red; Ivan direct call within 24h
Module deliverable not submitted at module end → Red regardless of confidence score
No pulse submission by Friday 5pm → Saturday morning auto-reminder text
Two consecutive missed submissions → Ivan direct call (not text)
Automation — H1
Pulse form → Airtable Weekly Pulse Log via Zapier (zero manual data entry)
Current session crosses S6, S11, S16, S20 → relevant JIT Loom kit auto-emails to teacher
Missed pulse by Friday 5pm → Saturday morning reminder (text or email TBD based on pilot)
Tier field auto-updates from pulse data via Airtable formula
Monday 8am weekly digest: all schools, current session, tier, open flags → Ivan email (Zapier)
Chipp Bot — H2
Bot replaces pulse form with organic daily teacher interaction. Bot reads student portfolio documents against rubric library and flags quality issues — not just completion. Admin bot delivers daily plain-language digest: "Roosevelt High is in M3, Insight Statements show themes not genuine insights — flagged." Ivan only calls Red sites; bot mediates everything else.
